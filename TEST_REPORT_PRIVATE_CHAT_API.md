## 修复记录

### 问题1: 数据库查询错误（严重）
**问题描述**: CRUD.read() 方法不支持 sort 和 limit 参数，导致查询消息时报错
```
Error: Can't both keep and omit fields except for _id
```

**影响范围**: 
- 获取聊天列表功能
- 获取消息列表功能
- 所有需要排序和限制的数据库查询

**修复方案**:
1. 修改 [db/crud.js](file:///d:/Desktop/bettermaoim/db/crud.js#L19-L63) 的 read 方法，支持 options 参数
2. 添加对 sort、limit、skip、projection 的支持
3. 使用 nedb-promises 的链式查询 API

**修复代码**:
```javascript
async read(conditions = {}, options = {}) {
  const { sort, limit, skip, projection } = options;
  let cursor = db.datastores[this.table].find(conditions, projection);
  
  if (sort) cursor = cursor.sort(sort);
  if (skip) cursor = cursor.skip(skip);
  if (limit) cursor = cursor.limit(limit);
  
  const docs = await cursor.exec();
  return Array.isArray(docs) ? docs : [];
}
```

**验证结果**: ✓ 修复成功，所有查询功能正常
